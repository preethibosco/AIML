Task 4


Perform normalization of all numeric attributes.

The following are features have number values.['StudentID', 'Age', 'Gender', 'Ethnicity', 'ParentalEducation', 'StudyTimeWeekly', 'Absences', 'Tutoring', 'ParentalSupport', 'Extracurricular', 'Sports', 'Music', 'Volunteering', 'GPA', 'GradeClass']
The dataset has all numberical values.
['StudentID'] is nominal unique identifier of data records and so not used in any model prediction.
['Gender','Tutoring', 'Extracurricular', 'Sports', 'Music', 'Volunteering',] are binary nominal categories so no normalization is required.
[ 'Ethnicity', 'ParentalEducation', 'ParentalSupport'] are categorical value and weuse one hot encoding to normalization them. and remove implicit order
Feature normalization is mostly needed to eliminate the effect of several quantitative features measured on different scales.
['Age','StudyTimeWeekly'] can only be normalised.we will use min max scaling to preserve the distribution
MinMaxScaler doesnt reduce the effect of outliers. but our data set feature ranges show that there are no outliers.so we can use minmax scaling

['GPA'] is target variable .A target variable with a large spread of values, in turn, may result in large error gradient values causing weight values to change dramatically, making the learning process unstable so requires scaling.but since GPA spread is less we neednot normalise.
['GradeClass'] is a linearly derived variable from target variable GPA.Normalisation of target variable is not required, based on above explanation
